## Findings considered and rejected

- **Unguarded `JSON.parse(localStorage…)` crashing the UI**: REJECTED after
  vetting. An initial grep listed every `JSON.parse(localStorage` site, but on
  reading each one (achievements.js:153, dashboard.js:1022/1061/1771/1782,
  game.js:338/612/667/683/701/723/1031/2806/2949, utils.js:115/219/268,
  account.js:63, minigame-acidbase.js:18, dashboard.js:1347/1494) every single
  site is wrapped in `try { … } catch`. The codebase consistently guards
  localStorage parsing. No bug. (So nobody re-audits it.)
- **Minify/bundle `style.css` (528KB) + `data/topics.js` (1.25MB)**: REJECTED —
  by-design. `CLAUDE.md` documents "SPA puro — sem build step" as an intentional
  decision; both assets are gzip-compressed by the CDN and Service-Worker cached
  after first load; `topics.js` is lazy-loaded. Revisit only if first-load
  metrics regress.
- **XSS via the 38 `innerHTML` sites in game.js**: REJECTED — user-controlled
  content (`player_name`, `character_name`) renders via `textContent`
  (leaderboard.js:318/398/408); the AI hint is escaped (`game.js:1986`,
  `escapeHtml(hint)`); feedback interpolates author-controlled question/explanation
  text from `data/topics.js`. CI also has an `innerHTML` XSS grep guard. No
  user-XSS vector found.
- **Edge Function security** (`mp-webhook`, `send-push`, `ai-mentor`): reviewed,
  no findings. HMAC + timing-safe compare + `processed_payments` idempotency on
  the webhook; admin-JWT-or-service-key gating on push; atomic server-side quota
  RPC with premium bypass on the AI mentor.
- **23 empty `catch {}` blocks in game.js**: not promoted — a smell, but no
  specific swallowed-error-on-a-critical-path was confirmed.
